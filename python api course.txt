Development Environment Setup:
-Install Python
-Install VS Code
-Setup Virtual Environment(Basically Like Docker, helps with keeping version consistency and prevents breaking from different versions)
    Two steps usually: VSCODE Selected Interpreter and The Terminal's Interpreter
    VSCODE Selected Interpreter:
    py -3 -m venv <name> : command for setting up a virtual environment in VSCODE.
    Afterwards select the Python intrepreter in the venv as the python exe from the command pallette
    Terminal's Interpreter:
    in the terminal, run the command:
    <current/project directory>/Script/activate or activate.bat for windows cmd/pwsh
    that command should let the terminal change the interpreter and if it works the venv's name should appear behind your prompt
    AFTER CLOSING VSCODE, do this or check to make sure that Vscode is pointing to the correct interpreter and also the terminal is pointing to the correct terminal
    Future or Past Me:
    "Use . test_venv/Scripts/activate to activate the venv for your project
     using ./test_venv/Scripts/activate creates a child process and is temporary so it does work but doesn't persist.
     Something to do with running a script in a new process versus using 'source' the '.' is the same as 'source'"
